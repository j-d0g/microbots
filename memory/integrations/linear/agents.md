## Workflow convention

1. **Create a Linear ticket first** — before opening any PR
2. Set priority: Urgent (blockers), High (sprint goals), Medium (normal), Low (nice-to-have)
3. Move to In Progress when starting
4. Link in GitHub PR description
5. Move to Done when PR merges

## Priority levels used

- **Urgent** — Production incidents, blockers
- **High** — Current sprint goals
- **Medium** — Backlog items
